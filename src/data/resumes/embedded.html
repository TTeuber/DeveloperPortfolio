Selected Projects

StompLink — two-microcontroller guitar pedal github

Stereo effects pedal (overdrive, tremolo, delay) with firmware split across two MCUs so realtime and bursty work never compete: a Daisy Seed (STM32H750 Cortex-M7) runs the audio at 48 kHz in interrupt context, while an ESP32-S3 (ESP-IDF v5.5, FreeRTOS) acts as USB host, WiFi gateway, and embedded web server, driving an SSD1306 OLED over I2C. A single ASCII protocol travels over both USB CDC and WebSocket, keeping six knobs, the OLED, and every connected browser in bidirectional sync, with parameter ownership arbitrated so web writes don't fight the hardware. The audio callback never takes a lock — parameters cross contexts as single-word volatile writes, de-zippered per sample — and the shared protocol header is host-testable C99/C++, run in CI on every commit.

pistomp-hal — hardware abstraction layer + simulator github

C++17 HAL for the pi-Stomp v3 pedal platform, consumed as a static library via CMake FetchContent. Hand-written drivers for the whole control surface — quadrature encoders (GPIO/libgpiod), footswitches via an MCP3008 SPI ADC, WS2812 LEDs, an ILI9341 TFT on LVGL v9 — plus a realtime duplex ALSA path (64-frame periods ≈1.3 ms, SCHED_FIFO, xrun recovery). The Board class owns the shared SPI bus mutex so consumers can't corrupt the bus, and a full macOS simulator backs the same headers, so apps compile unchanged for hardware or desktop — enforced by two-backend CI. Quadrature/debounce logic is header-only and Catch2-tested.

PistompPedalboard — live multi-effect pedalboard on Raspberry Pi 5 github

C++17 pedalboard I'm building as my own live performance rig — engineered to real-world reliability requirements, not as a demo. One binary drives 30+ effects, a Neural Amp Modeler block running amp captures in realtime, the on-device screen/encoders/footswitches through the HAL above, and a Svelte web control surface synced over Server-Sent Events. Chain edits are lock-free via epoch-based RCU, so effects reorder mid-performance without a click — verified by a TSan/ASan-swept stress test.

GestureSynth — polyphonic synthesizer plugin github

Juno-inspired subtractive synth built from scratch in C++20/JUCE 8, shipping as Standalone, AU, VST3, AUv3, and CLAP. All DSP hand-written: PolyBLEP anti-aliased 4×-oversampled oscillators, resonant state-variable filter, chorus, bucket-brigade-style delay, 16-line feedback-delay-network reverb. Real-time-safe throughout: lock-free UI↔audio messaging, cached atomics, no locks or allocations in processBlock. Catch2 tests/benchmarks, pluginval validation, cross-platform CI.
